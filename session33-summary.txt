### Issue Identified
**Order Submission Session Persistence:**
- Symptom: Clicking "Place Order" doesn't progress to confirmation page
- Root cause: Backend API returns `{"error":"Cart is empty"}` when POST /api/orders is called
- Context: Cart exists in frontend React state and displays correctly in cart drawer
- Impact: Backend session doesn't persist cart data from add-to-cart through checkout
- Severity: Medium - UI and flow work correctly, but final order creation fails
- Note: This appears to be a session cookie/credentials issue between frontend and backend
